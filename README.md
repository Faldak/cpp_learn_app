## 💡 Почему это приложение

Большинство тренажёров либо проверяют код «на глазок» через ИИ (и ошибаются),
либо объясняют сухо и непонятно. Здесь иначе:

- ✅ **Честная проверка** — задача засчитывается, только если код реально
  скомпилировался и выдал правильный результат.
- 🧠 **Глубокая теория** — каждый приём объясняется «как пятикласснику», по
  принципу «*мы пишем X, чтобы получить Y*», с образом для запоминания.
- 🇷🇺 **Полностью на русском** — интерфейс, уроки, подсказки и объяснения ошибок.

---

## 🎯 Главные принципы

| Принцип | Что это значит |
|---|---|
| **Правильность решает код** | Твоё решение компилирует настоящий компилятор (`g++`/`clang++`) и сравнивает вывод с эталоном. ИИ **никогда** не решает, верно или нет. |
| **ИИ — только объяснитель** | Включается лишь по твоей кнопке и только объясняет уже посчитанный результат или ошибку компилятора — всегда по-русски. Без ИИ приложение работает полностью. |
| **Всё на русском** | Каждый текст в интерфейсе и материалах — на русском, объяснения — для новичка без опыта. |
| **Двухуровневые ошибки** | Сначала — понятное объяснение из встроенного словаря (43 шаблона типичных ошибок), и только если не нашлось — необязательный ИИ. |

## 📦 Что внутри

- **8 блоков · 69 уроков · 603 задания** — от первой строчки `cout` до
  джуниор-проектов.
- **42 понятия C++** с глубокой, пошаговой теорией и 🧠-образами для запоминания.
- **Каждый урок самодостаточен**: всё, что нужно в заданиях, объяснено в его
  теории заранее — ученик нигде не «проваливается в пустоту».
- **Два типа заданий**: `output_match` (сверка вывода) и `sfml_rubric` (графика
  SFML — проверка по чек-листу + запуск).
- **Редактор кода** с подсветкой строк и кнопками «Проверить / Запустить /
  Сбросить / Подсказка / Показать решение».
- **Раздел «Прогресс»**: задания за сегодня, серия дней (streak), прогресс по
  курсу и блокам, частые ошибки.
- **Тёмная и светлая темы.** Весь прогресс хранится локально в SQLite.

## 🧰 Требования

- **Python 3.11+**
- **Компилятор C++** (обязательно): `g++` (MinGW-w64 на Windows, пакет `g++` на
  Linux) или `clang++` (Command Line Tools на macOS).
- **SFML** — необязательно, только для блока графики.
- **Ключ Groq API** — необязательно, только для ИИ-объяснений.

## ⚡ Установка и запуск

```bash
# 1. клонируем
git clone https://github.com/Faldak/cpp-learning-app.git
cd cpp-learning-app

# 2. (рекомендуется) виртуальное окружение
python -m venv .venv
# Windows:
.venv\Scripts\activate
# macOS / Linux:
source .venv/bin/activate

# 3. зависимости
pip install -r requirements.txt

# 4. запуск
python main.py
```

При первом запуске откроется экран настройки: проверка компилятора
(обязательно), SFML (можно позже) и ввод ключа Groq (можно пропустить).

<details>
<summary><b>Как установить компилятор C++</b></summary>

- **Windows:** MinGW-w64 через MSYS2 — `pacman -S mingw-w64-ucrt-x86_64-gcc`,
  затем добавь `g++` в `PATH`.
- **macOS:** `xcode-select --install`
- **Linux (Debian/Ubuntu):** `sudo apt install g++`
</details>

<details>
<summary><b>Как установить SFML (для блока графики)</b></summary>

- **Windows (MSYS2):** `pacman -S mingw-w64-ucrt-x86_64-sfml`
- **macOS:** `brew install sfml`
- **Linux (Debian/Ubuntu):** `sudo apt install libsfml-dev`
</details>

## 📦 Сборка в .exe / .app

Приложение можно собрать в один исполняемый файл с помощью Flet:

```bash
pip install flet[all]
flet pack main.py --name "C++ с нуля"
```

Готовый файл появится в папке `dist/`. Компилятор C++ у пользователя должен
быть установлен отдельно (приложение проверит его при первом запуске).

├── error_dictionary.py     # 43 шаблона понятных объяснений ошибок
├── ai_client.py            # необязательный ИИ-объяснитель (Groq)
├── progress_db.py          # локальный прогресс в SQLite
├── ui/                     # интерфейс (уроки, прогресс, настройки, темы)
├── content/                # ⚙️ ГЕНЕРИРУЕМЫЙ контент — не править вручную
│   ├── course_structure.json
│   ├── lessons/            # 69 уроков (теория + ссылки на задания)
│   └── tasks/              # 603 задания с эталонами и тестами
├── tools/                  # генератор контента и теории (см. ниже)
├── tests/                  # автотесты
├── installers/             # вспомогательные скрипты установки

> ⚠️ Файлы в `content/` **генерируются** скриптами из `tools/`. Не редактируй их

> запуска — в [`tools/BANK_PIPELINE.md`](tools/BANK_PIPELINE.md).

## 🔍 Как устроена проверка задач

1. Ты пишешь код в редакторе и жмёшь **«Проверить»**.
2. `compiler_service.py` компилирует его настоящим `g++`/`clang++` (стандарт
   C++17) во временной песочнице.
3. Программа запускается с заранее заданными входными данными.
4. `task_checker.py` сравнивает фактический вывод с эталонным — **строго и
   детерминированно**.
5. Если есть ошибка компиляции — сначала ищем понятное объяснение в словаре из
   43 шаблонов; ИИ подключается только если шаблон не найден и только по твоей
   кнопке.

**ИИ ни на одном шаге не решает, правильный ответ или нет** — он лишь объясняет
уже посчитанный результат человеческим языком.
